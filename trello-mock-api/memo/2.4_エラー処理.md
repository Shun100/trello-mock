# エラー処理

## 同期コードでのエラー処理

```TypeScript
  app.get('/sync-error', (req, res) => {
    // 同期コードでのエラーは自動的にキャッチされる
    throw new Error('同期コードでのエラー');

    res.send('正常なレスポンス'); // この行は実行されない
  });
```

## 非同期コードでのエラー処理

- 非同期処理の内部で発生したエラーは、Expressは補足できない
- そのため、開発者が明示的に`catch`ブロックなどでエラーを捕まえ、`next(err)`のようにエラーオブジェクトを`next`関数に渡してあげる必要がある

```TypeScript
  // Promiseの場合
  app.get('/async-promise', (req, res, next) => {
    fetchDataPromise();
      .then(data => res.json(data))
      .catch(err => next(err)); // 自分で補足して渡す
  });

  // async/awaitの場合
  app.get('/async-await', async (req, res, next) => {
    try {
      const data = await fetchDataAsync();
      res.json(data);
    } catch (err) {
      next(err); // 自分で補足して渡す
    }
  });
```

## グローバルエラーハンドラ（ミドルウェア）

- アプリケーション全体のエラーを一元的に処理するために、ミドルウェアの最後にエラーハンドリングミドルウェアを定義する
- これにより、いずれかのルートやミドルウェアで`next(err)`が呼び出された場合や、同期コードで例外が発生した場合エラーハンドラーが実行される

```TypeScript
  // 通常のミドルウェアと異なり、パラメータが4つ必要なので注意
  app.use((err, req, res, next) => {
    err.statusCode = err.statusCode ?? 500;
    err.message = err.message ?? 'サーバエラーが発生しました';
  });

  // エラーをロギング
  console.error(`[${new Date().toISOString()}] ${req.method} ${req.path} - Error: ${err.message}`);
  if (process.NODE_ENV === 'development') {
    console.error(err.stack);
  }

  res.status(res.statusCode.json({ message: err.message }));
```

## 404エラーの処理

- どのルートにも一致しないリクエストは上記のエラーハンドラーに到達しないため、404を処理するためのミドルウェアをエラーハンドラーの直前に配置する

```TypeScript
  app.use(req, res, next) => {
    const error = new Error(`ルートが見つかりません: ${req.originalUrl}`);
    error.status = 404;
    next(error);
  }

  // この後にグローバルエラーハンドラを定義
  // app.use((err, req, res, next) => {...});
```
